## ✨ Идея проекта

Это не обычный бот с лентой ответов. В каждом чате бот поддерживает **одно управляющее сообщение**: кнопки, live-данные, результаты операций и системные события редактируют его на месте. Чат остаётся чистым, а пользователь всегда видит актуальный экран.

| Что | Как работает |
| --- | --- |
| 🖥️ Интерфейс | Один редактируемый экран на чат с inline-кнопками |
| 🔄 Live-данные | Баланс, позиции и рынок обновляются каждые 2 секунды |
| 🧪 Безопасный старт | `DRY_RUN=True` по умолчанию: торговые POST-запросы симулируются |
| 👥 Несколько пользователей | Личные алерты, настройки и журналы разделены по Telegram chat ID |
| 🛡️ Общий аккаунт | Только администраторы получают доступ к реальному торговому контуру |

## 🚀 Возможности

### 🧭 Telegram UX

- **Единый экран** — бот редактирует текущее сообщение вместо отправки цепочки новых.
- **Живая навигация** — переход на другой экран отменяет предыдущую задачу обновления, поэтому старые данные не перезаписывают новый экран.
- **Защита от зависших кнопок** — каждый callback подтверждается; кнопки устаревших сообщений ведут на актуальное меню.
- **Безопасная обработка медленных операций** — запросы Bybit, DeepSeek, SQLite и внешних сервисов выполняются вне event loop.
- **Авто-события без спама** — уведомления автоторговли и алертов редактируют экран и объединяются при всплеске событий.

### 💼 Аккаунт и позиции

- 💰 Баланс Unified Account: wallet balance, equity, свободные средства, маржа позиций и ордеров.
- 📊 Открытые позиции: направление, размер, цена входа, текущая цена, PnL и ROI.
- 🔍 Детальный экран позиции с stop-loss, take-profit, плечом и расчётными метриками.
- ❌ Закрытие одной позиции или всех позиций с отдельным подтверждением.
- 📜 История закрытого PnL из Bybit.
- 🧪 В `DRY_RUN` закрытие и создание ордеров отображаются как симуляция и не уходят на биржу.

### 📈 Аналитика и AI

- Технические индикаторы: **EMA, RSI, MACD и ATR**.
- Анализ нескольких таймфреймов: 3 минуты, 5 минут, 1 час и 4 часа.
- AI-рекомендации DeepSeek в строгом JSON-формате с валидацией чисел и токенов.
- Кэширование тяжёлых индикаторов: цена остаётся live, а свечные расчёты не перезапрашиваются каждые 2 секунды.
- 🌍 Обзор рынка: капитализация, объём за 24 часа, доминация BTC и трендовые активы.

### 🤖 Авто-режим и риск

- Открытие сделок только для разрешённых токенов и с учётом минимального размера лота.
- Округление объёма через `Decimal` по шагу инструмента.
- Расчёт фактического риска до stop-loss, а не только номинальной суммы ордера.
- Проверка risk/reward, ликвидационного буфера, максимального риска на сделку и общего риска портфеля.
- Блокировка новых сделок, если уже есть незащищённая позиция без stop-loss.
- Защита от пирамидинга в ту же сторону и корректное закрытие противоположной позиции.

### 🔔 Личные алерты

- 💲 Ценовые алерты и 📊 RSI-алерты.
- Направление: значение **выше** или **ниже** заданного порога.
- Режимы: **один раз** или **повторяемый**.
- Cooldown между срабатываниями.
- Срабатывание только при **реальном пересечении** уровня: алерт не шумит сразу после создания, если цена уже находится за границей.
- Персональный актив и RSI-интервал выбираются в профиле алертов.
- Каждое создание, удаление и срабатывание отражается в локальном журнале.

### 🧾 Журнал и роли

- Журнал фиксирует создание/удаление/срабатывание алертов, остановку и запуск авто-режима, а также запросы закрытия позиций.
- `ADMIN_TELEGRAM_IDS` разделяет пользователей: владельцы управляют общим Bybit-аккаунтом; остальные используют алерты, обзор рынка и личный журнал.
- Это важно: несколько пользователей не получают отдельные субсчета — торговля всегда относится к одному подключённому биржевому аккаунту.

```text
TgBot-Bybot-control/
├── api/                         # Bybit V5 и DeepSeek-клиенты
│   ├── bybit_api.py             # Подпись запросов, retry, dry-run
│   ├── deepseek_api.py          # AI-анализ
│   └── tg_notify.py             # События через единый экран
├── core/                        # Бизнес-логика
│   ├── auto_trading.py          # Исполнение сигналов и контроль риска
│   ├── alerts.py                # Пересечения порогов
│   ├── alert_scheduler.py       # Один async-планировщик
│   ├── market_data.py           # Индикаторы и свечи
│   ├── market_overview.py       # Внешний обзор рынка
│   └── prompt_builder.py        # Контракт AI-промпта
├── storage/
│   └── database.py              # SQLite-репозиторий и схема
├── telegram_bot/
│   ├── ui.py                    # Редактирование одного сообщения
│   ├── activity_middleware.py   # Профили и контроль ролей
│   ├── handlers/                # Экраны Telegram
│   └── keyboards/               # Inline-клавиатуры
├── utils/                       # Расчёты, форматирование, логирование
├── config.py                    # Безопасные defaults без секретов
├── requirements.txt
└── main.py                      # CLI-старт
```

В проекте нет старых параллельных слоёв `features`, `services`, `tasks` и нет второго thread-based планировщика. Работают один event loop, один Bybit-адаптер и один путь торгового исполнения.

## 🗃️ База данных и масштабирование

По умолчанию бот создаёт SQLite-файл `data/crypto_bot.sqlite3`; он игнорируется Git.

| Таблица | Что хранит |
| --- | --- |
| `users` | Telegram-профиль, роль, статус, локаль, часовой пояс, настройки уведомлений, экран, актив и интервал по умолчанию |
| `alerts` | Тип, символ, порог, направление, интервал, повторяемость, cooldown, последнее значение и число срабатываний |
| `activity_log` | Личные действия, события алертов и системные записи |

### Почему SQLite

- ✅ Надёжное постоянное хранилище без отдельного сервера.
- ✅ WAL-режим, внешние ключи, транзакции и ожидание блокировки.
- ✅ Хороший выбор для одного процесса Telegram-бота и обычной многопользовательской нагрузки.
- ➡️ PostgreSQL нужен при нескольких одновременно работающих экземплярах, репликации или существенно более высокой параллельности.
- ➡️ Redis не является источником данных для алертов; в будущем он может быть добавлен только для кэша или распределённых блокировок.

UI и доменная логика не пишут SQL напрямую, поэтому хранилище можно заменить без переписывания Telegram-экранов.

## ⚙️ Установка и запуск

Требуется **Python 3.10+**.

```powershell
pip install -r requirements.txt
python main.py
```

Режимы запуска:

```powershell
python main.py telegram  # Telegram-бот
python main.py auto      # Авто-режим из консоли
python main.py --help    # Справка
```

### 🔐 Переменные процесса

Проект **не создаёт и не загружает `.env`**, не использует `config.json` и не хранит ключи в Git.

| Переменная | Для чего нужна |
| --- | --- |
| `TELEGRAM_TOKEN` | Обязательна для запуска Telegram-бота |
| `BYBIT_API_KEY`, `BYBIT_API_SECRET` | Приватные данные Bybit и live-торговля |
| `DEEPSEEK_API_KEY` | AI-рекомендации |
| `ADMIN_TELEGRAM_IDS` | Telegram user ID владельцев через запятую |
| `DRY_RUN` | `True` по умолчанию; `False` разрешает реальные POST-запросы |
| `CRYPTO_DB_PATH` | Необязательный путь к SQLite-файлу |

> ⚠️ Без `ADMIN_TELEGRAM_IDS` торговые кнопки специально недоступны всем пользователям. Это предотвращает доступ случайного пользователя к общему биржевому аккаунту.

## 🛡️ Безопасность и ограничения

- Не передавайте API-ключи в чат, commit или screenshot.
- Используйте минимальные разрешения Bybit API: чтение и торговля, без вывода средств.
- Сначала проверьте сценарии в `DRY_RUN=True`.
- `DRY_RUN=False` означает реальный торговый POST-запрос; ответственность за торговые решения остаётся у владельца аккаунта.
- Фоновый алерт никогда не создаёт новое сообщение. Если старый экран удалён вручную, новый экран появится только после следующего действия пользователя.
- Ввод порога алерта оставляет в чате пользовательское сообщение Telegram — это ограничение платформы; ответ бота всё равно редактирует единый экран.

## 🧪 Проверка изменений

В репозиторий не добавлены production-тесты. Перед публикацией выполняются буферные проверки:

- компиляция Python-модулей;
- импорт Telegram-бота;
- проверка схемы SQLite;
- два изолированных пользователя;
- одноразовые и повторяемые пересечения алертов;
- `git diff --check`.
