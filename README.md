# Crypto Trading Bot for Bybit

Telegram-бот для контроля фьючерсного аккаунта Bybit: позиции, баланс, AI-анализ, управляемая автоторговля, личные ценовые и RSI-алерты, обзор рынка и журнал действий.

## Главные принципы

- Один чат — одно редактируемое сообщение бота. Кнопки, live-экраны, события автоторговли и алерты изменяют его, а не засоряют историю новыми сообщениями.
- Данные баланса, позиций и цены обновляются в том же сообщении каждые 2 секунды. Медленные запросы Bybit, AI и внешних сервисов вынесены из event loop.
- Сделки в режиме `DRY_RUN=True` только симулируются. В live-режиме перед открытием позиции проверяются размер, риск до stop-loss, RR, ликвидационный буфер и суммарный риск.
- Один Bybit-аккаунт является общим торговым контуром. Пользовательские данные — алерты, экран, журнал и предпочтения отображения — изолированы по Telegram chat ID.

## Возможности

- Баланс Unified Account, позиции, PnL, история закрытых сделок и закрытие позиций.
- Рыночная аналитика: EMA, RSI, MACD, ATR и AI-рекомендации DeepSeek.
- Авто-режим с ограничением риска, защитой стоп-лоссом и запретом незащищённых позиций.
- Личные алерты цены и RSI с направлением, режимом «один раз» / «повтор», cooldown и срабатыванием только на пересечении порога.
- Обзор капитализации, объёма, доминации BTC и трендовых активов.
- Журнал пользовательских и системных действий.

## База данных и многопользовательский режим

По умолчанию используется SQLite: `data/crypto_bot.sqlite3`. Файл базы не попадает в Git.

SQLite выбран намеренно: для одного процесса Telegram-бота он даёт транзакции, WAL-режим и устойчивое хранение без отдельной инфраструктуры. Redis не подходит для постоянных алертов и журнала — он мог бы быть только кэшем или распределённой блокировкой. PostgreSQL понадобится, когда появятся несколько одновременно работающих экземпляров бота, репликация или высокая параллельная нагрузка.

Схема хранит:

- `users`: Telegram-профиль, статус, локаль/часовой пояс, настройки уведомлений, символ и RSI-интервал по умолчанию, ограничения интерфейса, ID единого экранного сообщения и служебные метки;
- `alerts`: тип, символ, интервал, направление, порог, повторяемость, cooldown, последнее значение и историю срабатываний;
- `activity_log`: действия с позициями и авто-режимом, создание/удаление/срабатывание алертов и системные события.

Переход к PostgreSQL не потребует менять Telegram- или торговые обработчики: они обращаются к хранилищу через его методы, а не пишут SQL в UI. Redis и PostgreSQL намеренно не включены в зависимости, пока под них нет поднятой и проверенной инфраструктуры.

## Запуск

Нужен Python 3.10+.

```powershell
pip install -r requirements.txt
python main.py
```

Секреты не хранятся ни в репозитории, ни в JSON. Бот читает ключи только из переменных окружения процесса:

- `TELEGRAM_TOKEN` — обязателен для Telegram-режима;
- `BYBIT_API_KEY`, `BYBIT_API_SECRET` — нужны для приватных данных и live-торговли;
- `DEEPSEEK_API_KEY` — нужен только для AI-рекомендаций.
- `ADMIN_TELEGRAM_IDS` — список Telegram user ID владельцев через запятую. Только они видят и управляют общим биржевым контуром: балансом, позициями, сделками и авто-режимом. Остальные пользователи могут пользоваться личными алертами, обзором рынка и журналом.

Создавать `.env` не требуется и проект его не загружает. По умолчанию включён безопасный `DRY_RUN=True`. Для реальной торговли установите `DRY_RUN=False` в окружении процесса только после проверки лимитов риска.
Без `ADMIN_TELEGRAM_IDS` торговые кнопки намеренно недоступны всем пользователям.

## Навигация в Telegram

`/start` или `/menu` открывает главный экран. Из него доступны:

- **Позиции / Баланс** — live-экраны с обновлением раз в 2 секунды;
- **AI-рекомендации / Анализ рынка** — актуальный технический разбор;
- **Алерты** — создание и удаление личных ценовых и RSI-алертов;
- **Обзор рынка** — кэшируемый read-only обзор внешнего рынка;
- **Сделки / Журнал** — биржевая история и локальная история действий;
- **Авто-режим / Настройки** — контроль автоматики и персональные настройки алертов.

При вводе порога алерта пользовательское сообщение Telegram, конечно, остаётся в чате, но бот не создаёт отдельный ответ: результат редактирует единственный экран.

## Структура

```text
api/             Bybit V5 и DeepSeek адаптеры
core/            риск, автоторговля, индикаторы, алерты, обзор рынка
storage/         SQLite-репозиторий и схема данных
telegram_bot/    единый экран, middleware, handlers, keyboards
utils/           расчёты, форматирование, логирование
```

Устаревшие слои `features`, `services`, `tasks` и старые обработчики предыдущего бота удалены. В приложении остался один event loop и один фоновой планировщик алертов.

## Ограничения и безопасность

- Один Telegram пользователь не получает отдельный торговый субаккаунт: все операции затрагивают общий подключённый Bybit-аккаунт.
- Алерт, созданный когда цена уже за уровнем, сначала только запомнит текущее значение. Он сработает при следующем реальном пересечении.
- Повторяемый алерт снова сработает только после возврата значения за границу и нового пересечения; cooldown защищает от частых уведомлений.
- Если старое экранное сообщение удалено вручную, бот создаст замену только при следующем пользовательском действии. Фоновый алерт никогда не создаёт новое сообщение.

## Разработка

В проект не добавлены production-тесты. Перед публикацией выполняются буферная проверка импорта, компиляция и локальная проверка схемы SQLite/логики пересечения без обращения к Telegram или бирже.
